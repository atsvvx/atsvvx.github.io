WSL2 体验

我在 WSL 里面改不了 Folder 名, 所以我去了 VsCode 的 Iusse 上, 发现官方也说改不了, 要想改名, 你需要将 WSL 升级为 WSL2. 当热, WSL2 只有在 Windows Insider 中才有, 这意味着你还需要升级你的 Windows.

危险

使用 WSL2 就不要尝试去访问 /mnt/* 不要问为什么? 巨硬为了解决都慢做出了牺牲, 让 WSL 内部快了起来, 同时 /mnt/* 的 I/O 更慢了.

为什么升级 WSL2

I see EACCESS: permission denied error trying to rename a folder in the open workspace这个问题我也遇到了.

怎么升级 WSl2

  1. 升级 Windows -> Insider
  2. 安装 WSL1
  3. 升级 WSl2

升级 Windows -> Insider

于是, 我照着官方的指引一步一步操作, 首先申请体验哟用户, 但是这个功能应该在简体版的 Windows 有大问题, 所以我选择了下载镜像的方式, 安装过程和新安装一个 Windows 的过程差不多, 这里不多赘述了. (_当然大家可以试试教程)

安装 WSL1

Enable-WindowsOptionalFeature -Online -FeatureName VirtualMachinePlatform
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux

打开 AppStore 搜索 WSL 没有任何骚操作.

升级 WSL2

wsl --set-version Ubuntu 2

上述过程, 垃圾场的固态跑了 5 分钟.

速度体验

一开始没看官方的说明, 将 github 上的项目部署到了 Windows 的用户文件夹下

npm i

慢的有种想装回来原来的 Windows 的感觉, 和微软说的大幅提升 IO 性能完全不相符, 最后搁网上看了一下, 才发现是提升了除 /mnt/* 外的所有文件, 换到 /home/svvx 里面, 快的一批.
